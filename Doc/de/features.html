Detaillierte Funktionen

1. Modellbearbeitung und Verschmelzung

Das Tool ermöglicht das gleichzeitige Laden von bis zu drei Modellquellen:

  • Modell A (GGUF): Basismodell.
  • Modell B (GGUF): Zweites Modell zum Verschmelzen (Merge).
  • Modell S (Safetensors): Integration von PyTorch-Gewichten.

Sie können spezifische Tensoren oder nach Ebenen (alle "blks", oder ungerade/gerade Schichten) in das Ausgangsmodell übertragen.

2. Visualisierung und Analyse

Greifen Sie auf die Ansicht "Tensoren" zu, um das Innere Ihrer Modelle zu untersuchen.

  • Zeitreihen (Plots): Visualisieren Sie Float-Werte entlang eines Tensors. Erkennen Sie Anomalien und Ausreißer.
  • Vergleich (T1 - T2): Subtrahieren Sie zwei Tensoren, um den Unterschied zu sehen (nützlich für die Analyse von Merges).
  • Histogramme: Analysieren Sie die Gewichtsverteilung (normal, gaußförmig, usw.).

3. Namens-Mapping

Die Mapping-Engine ermöglicht das automatische Umbenennen von Tensoren, um ein Modell an eine Zielarchitektur anzupassen.

  • Unterstützung für generische Muster (z.B. `blk.{}.attn.weight`).
  • Externe Konfigurationsdateien (.txt).
  • Verwaltung von ignorierten Präfixen.

4. Intelligente Quantisierung

Echtzeit-Konvertierung von Tensor-Typen:

  • Llama.cpp DLL: Nutzung nativer Bibliotheken für hochpräzise Dequantisierung (minimierter RMSE).
  • Impl-Modus: Alternativem Software-Algorithmus für maximale Stabilität oder wenn die DLL nicht verfügbar ist.
  • Quantisierungssimulation: Sehen Sie, wie ein FP32-Tensor wie Q4_K aussehen würde, ohne ihn tatsächlich zu schreiben.

5. Split und Merge

Integrierte Tools für:

  • Split: Aufteilen eines großen GGUF-Modells in mehrere Dateien (z.B. 4GB).
  • Merge: Wiedervereinigen mehrerer GGUF-Shards zu einer einzigen kohärenten Datei.
