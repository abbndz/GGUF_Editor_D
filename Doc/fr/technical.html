Architecture Technique

Documentation destinée aux développeurs et contributeurs.

Technologies Utilisées

  • Langage : Delphi 10/11/12 (Object Pascal).
  • GUI : VCL (Visual Component Library).
  • Graphique : TeeChart (pour les charts complexes).
  • FFI : Chargement dynamique de DLL C/C++ (ggml-base.dll).

Structure des Modules

Fichier Unit Rôle
uGGUFModel.pas Définition des classes de données (Tensors, KVs).
uGGUFReader.pas Lecture binaire des fichiers GGUF.
uGGUFWriter.pas Écriture et manipulation des flux de sortie.
uGgmlQuants.pas Interface avec les fonctions de quantification (DLL ou Impl).
uViewTensors.pas Logique de visualisation, détection d'anomalies et graphiques.
uTensorTranspose.pas Logique de transposition de matrices (pour Safetensors).

Format de Mapping

Les fichiers de mapping sont des fichiers texte (.txt) contenant des règles pattern=replacement.


model.layers.0.input_layernorm.weight -> attention.input_layernorm
model.layers.{}.feed_forward.w1.weight -> mlp.gate

Gestion de la Mémoire

L'application utilise une approche "Lazy Load" pour les données des tenseurs. Les données brutes ne sont décodées en mémoire (F32) qu'au moment de l'affichage dans les graphiques ou de la quantification, afin de minimiser l'empreinte RAM sur les très gros modèles.
