Детальные возможности

1. Редактирование и объединение моделей

Инструмент позволяет загружать до трех источников моделей одновременно:

  • Модель A (GGUF): Базовая модель.
  • Модель B (GGUF): Вторая модель для объединения (merge).
  • Модель S (Safetensors): Интеграция весов PyTorch.

Вы можете переносить отдельные тензоры или целые слои (все "blks" или четные/нечетные слои) в выходную модель.

2. Визуализация и анализ

Доступ к представлению "Тензоры" для инспекции внутренних данных ваших моделей.

  • Временные ряды (Графики): Визуализация значений float вдоль тензора. Обнаружение аномалий и выбросов.
  • Сравнение (T1 - T2): Вычитание двух тензоров для просмотра разницы (полезно для анализа слияния).
  • Гистограммы: Анализ распределения весов (нормальное, гауссовское и т.д.).

3. Маппинг имен (Name Mapping)

Движок маппинга позволяет автоматически переименовывать тензоры для адаптации модели к целевой архитектуре.

  • Поддержка общих шаблонов (например, `blk.{}.attn.weight`).
  • Внешние файлы конфигурации (.txt).
  • Управление игнорируемыми префиксами.

4. Интеллектуальное квантование

Конвертация типов тензоров в реальном времени:

  • DLL Llama.cpp: Использование нативных библиотек для высококачественного деквантования (минимизация RMSE).
  • Режим Impl: Альтернативный программный алгоритм для максимальной стабильности или когда DLL недоступна.
  • Симуляция квантования: Посмотрите, как будет выглядеть FP32 тензор после конвертации в Q4_K, не записывая его.

5. Разделение и Объединение (Split and Merge)

Интегрированные инструменты для:

  • Разделение (Split): Разделение большой GGUF модели на несколько файлов (например, по 4 ГБ).
  • Объединение (Merge): Сборка нескольких фрагментов GGUF в один согласованный файл.
